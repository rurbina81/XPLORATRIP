## FLUJO DE TRABAJO RECOMENDADO PARA ROMER

### Diario

1. **Mañana**: revisar WhatsApp. Usar atajos de `AutoReplyWhatsApp.md` (`/vig`, `/venc`, `/ced`, etc.) según el tipo de lead.
2. **Tarde**: actualizar `LeadTracker` con leads nuevos del día. Si hay leads que pasaron 2 días sin responder, mandar Mensaje #1 de `DripSequence`.

### Lunes

1. Recibir automáticamente `Brief lunes` en `_Contexto/Briefs/`.
2. Definir los 3 to-dos de la semana.
3. Confirmar piezas de contenido a publicar según `ContentBrief` vigente.

### Miércoles

1. Mandar broadcast WhatsApp con el copy del ContentBrief.
2. Publicar pieza del día (reel o carrusel).

### Viernes

1. Recibir automáticamente `Cierre viernes`.
2. Llenar `WeeklyMetricsDashboard` (toma 5-7 minutos).
3. Definir empuje de la próxima semana.

### Cada quincena

1. Revisar `ContentBrief` vigente, contar piezas publicadas vs planeadas.
2. Pedir a Claude un nuevo `ContentBrief` para las próximas 2 semanas (`/content-strategy` o "qué publico").

### Cada trimestre

1. Correr `/quarterly-review` con datos acumulados.
2. Actualizar `BusinessContext.md` si cambió subsegmento dominante, ciudades, dolor.

## DÓNDE PEDIR AYUDA A CLAUDE

| Necesidad | Cómo pedir |
|---|---|
| Tengo un lead nuevo, ayúdame | "Tengo un lead, ayúdame a clasificarlo" o pega el mensaje |
| Qué publico esta semana | "Qué publico" o "Plan de contenido" |
| Hazme un flyer / reel / carrusel | "Hazme un flyer de [tema]" o "Genera los assets" |
| Responde a este cliente | "Respóndele a este cliente: [pega el mensaje]" |
| Cómo va el negocio | "Cómo va el negocio" o "Pulso del negocio" |
| Análisis de precios | "¿Estoy ganando con esta ruta?" o "Análisis de margen" |
| Flujo de caja | "Flujo de caja" o "¿Llegamos al mes?" |
| Resumen del trimestre | "Resumen del Q" o "QBR" |

El router (`smb-router`) entiende lenguaje natural — no necesitas memorizar comandos.
